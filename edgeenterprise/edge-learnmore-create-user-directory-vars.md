<span data-ttu-id="b21a6-104">この記事では、Microsoft Edge を変更するときにハードコードされたパスを使用する代わりに、データ ディレクトリ変数を使用する方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="b21a6-104">This article explains how you can use data directory variables instead of using hard-coded paths when modifying Microsoft Edge.</span></span>

## <a name="path-variables"></a><span data-ttu-id="b21a6-106">パス変数</span><span class="sxs-lookup"><span data-stu-id="b21a6-106">Path variables</span></span>

<span data-ttu-id="b21a6-107">データ ディレクトリ パスを変更するためのポリシー ([UserDataDir](microsoft-edge-policies.md#userdatadir) や [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) の構成など) では、変数がサポートされています。</span><span class="sxs-lookup"><span data-stu-id="b21a6-107">Policies for modifying data directory paths (For example, configuring the [UserDataDir](microsoft-edge-policies.md#userdatadir) or [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) support variables.</span></span> <span data-ttu-id="b21a6-108">これらのポリシーを構成するときは、ハードコードされたパスではなく、変数を使用できます。</span><span class="sxs-lookup"><span data-stu-id="b21a6-108">When configuring these policies, you can use variables instead of hard-coded paths.</span></span> <span data-ttu-id="b21a6-109">たとえば、既定の場所ではなく、Windows ユーザーの "ローカル アプリケーション データ" にプロファイル データを格納する場合は、</span><span class="sxs-lookup"><span data-stu-id="b21a6-109">For example, to store your profile data under user local application data on Windows instead of the default location.</span></span> <span data-ttu-id="b21a6-110">[UserDataDir](microsoft-edge-policies.md#userdatadir) ポリシーを **${local_app_data}\Edge\Profile** に設定します。</span><span class="sxs-lookup"><span data-stu-id="b21a6-110">Set the [UserDataDir](microsoft-edge-policies.md#userdatadir) policy to **${local_app_data}\Edge\Profile**.</span></span> <span data-ttu-id="b21a6-111">ほとんどの Windows 10 インストールでは、このパスが *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile* に解決されます。</span><span class="sxs-lookup"><span data-stu-id="b21a6-111">On most Windows 10 installations, this path resolves to *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile*.</span></span>

>[!NOTE]
><span data-ttu-id="b21a6-112">現在の**プロファイル パス**を確認するには、**[バージョン情報]** ページを開きます ("edge://version" と入力します)。</span><span class="sxs-lookup"><span data-stu-id="b21a6-112">To view the current  **Profile path**, open the **About version** page (type "edge://version").</span></span> <span data-ttu-id="b21a6-113">**プロファイル パス**には *C:\Users\\&lt;現在のユーザー&gt;\AppData\Local\Microsoft\Edge\User Data\Default* という形式を使用します。</span><span class="sxs-lookup"><span data-stu-id="b21a6-113">The **Profile path** follows this format: *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\User Data\Default*.</span></span>

### <a name="guidance-for-using-path-variables"></a><span data-ttu-id="b21a6-114">パス変数を使用するためのガイダンス</span><span class="sxs-lookup"><span data-stu-id="b21a6-114">Guidance for using path variables</span></span>

<span data-ttu-id="b21a6-115">パスに変数を使用する前に、次のガイダンスをご確認ください。</span><span class="sxs-lookup"><span data-stu-id="b21a6-115">Review the following guidance before using variables for paths.</span></span>

- <span data-ttu-id="b21a6-116">Microsoft Edge によってさまざまなデータが格納されるパスを決定するすべてのポリシーは、プラットフォームによって異なります。</span><span class="sxs-lookup"><span data-stu-id="b21a6-116">All policies that involve paths where Microsoft Edge stores different data are platform dependent.</span></span> <span data-ttu-id="b21a6-117">これらのポリシーには、特定のプラットフォームでのみ使用できるものと、すべてのプラットフォームで使用できるものがあります。</span><span class="sxs-lookup"><span data-stu-id="b21a6-117">Some of these policies are available only on specific platforms, but others can be used on all platforms.</span></span>
- <span data-ttu-id="b21a6-118">さまざまな場所から起動されるアプリケーションによって発生するエラーを回避するには、必ず絶対パスを指定してください。</span><span class="sxs-lookup"><span data-stu-id="b21a6-118">To avoid errors caused by applications starting from different locations on different occasions, make sure that paths are absolute.</span></span>
- <span data-ttu-id="b21a6-119">変数はすべて、1 つのパス内で 1 回のみ指定できます。</span><span class="sxs-lookup"><span data-stu-id="b21a6-119">Every variable can occur only once in a path.</span></span> <span data-ttu-id="b21a6-120">ほとんどの場合、変数は絶対パスに解決されるため、変数を使用するにはこの方法を使用する必要があります。</span><span class="sxs-lookup"><span data-stu-id="b21a6-120">For most of them, this is the only meaningful way to use variables, because they resolve to absolute paths.</span></span>
- <span data-ttu-id="b21a6-121">ほとんどのポリシーでは、パスが存在しなければ作成されます (既存の環境で可能な場合)。</span><span class="sxs-lookup"><span data-stu-id="b21a6-121">Almost all policies will create the path if it doesn't exist (if possible in the existing circumstances).</span></span>
- <span data-ttu-id="b21a6-122">Microsoft Edge のバージョン/チャネルによってフォルダー構造の処理方法が異なるため、ポリシーによっては、ネットワークの場所を使用すると、予期しない結果を招く場合があります。</span><span class="sxs-lookup"><span data-stu-id="b21a6-122">Using network locations for some policies can lead to unexpected results due to differences in how different versions/channels of Microsoft Edge handle the folder structure.</span></span>
