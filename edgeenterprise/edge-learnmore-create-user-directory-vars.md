# <span data-ttu-id="6b32f-103">Microsoft Edge ユーザー データ ディレクトリ変数を作成する</span><span class="sxs-lookup"><span data-stu-id="6b32f-103">Create Microsoft Edge user data directory variables</span></span>

<span data-ttu-id="6b32f-104">この記事では、Microsoft Edge を変更するときにハードコードされたパスを使用する代わりに、データ ディレクトリ変数を使用する方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="6b32f-104">This article explains how you can use data directory variables instead of using hard-coded paths when modifying Microsoft Edge.</span></span>

>[!NOTE]
><span data-ttu-id="6b32f-105">この記事は、Microsoft Edge version 77 以降に適用されます。</span><span class="sxs-lookup"><span data-stu-id="6b32f-105">This article applies to Microsoft Edge version 77 or later.</span></span>

## <span data-ttu-id="6b32f-106">パス変数</span><span class="sxs-lookup"><span data-stu-id="6b32f-106">Path variables</span></span>

<span data-ttu-id="6b32f-107">データ ディレクトリ パスを変更するためのポリシー ([UserDataDir](microsoft-edge-policies.md#userdatadir) や [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) の構成など) では、変数がサポートされています。</span><span class="sxs-lookup"><span data-stu-id="6b32f-107">Policies for modifying data directory paths (For example, configuring the [UserDataDir](microsoft-edge-policies.md#userdatadir) or [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) support variables.</span></span> <span data-ttu-id="6b32f-108">これらのポリシーを構成するときは、ハードコードされたパスではなく、変数を使用できます。</span><span class="sxs-lookup"><span data-stu-id="6b32f-108">When configuring these policies, you can use variables instead of hard-coded paths.</span></span> <span data-ttu-id="6b32f-109">たとえば、既定の場所ではなく、Windows ユーザーの "ローカル アプリケーション データ" にプロファイル データを格納する場合は、</span><span class="sxs-lookup"><span data-stu-id="6b32f-109">For example, to store your profile data under user local application data on Windows instead of the default location.</span></span> <span data-ttu-id="6b32f-110">[UserDataDir](microsoft-edge-policies.md#userdatadir) ポリシーを **${local_app_data}\Edge\Profile** に設定します。</span><span class="sxs-lookup"><span data-stu-id="6b32f-110">Set the [UserDataDir](microsoft-edge-policies.md#userdatadir) policy to **${local_app_data}\Edge\Profile**.</span></span> <span data-ttu-id="6b32f-111">ほとんどの Windows 10 インストールでは、このパスが *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile* に解決されます。</span><span class="sxs-lookup"><span data-stu-id="6b32f-111">On most Windows 10 installations, this path resolves to *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile*.</span></span>

>[!NOTE]
><span data-ttu-id="6b32f-112">未設定の場合、Microsoft Edge では既定のディレクトリ パスが使用されます。</span><span class="sxs-lookup"><span data-stu-id="6b32f-112">If left unset, Microsoft Edge uses default directory paths.</span></span> <span data-ttu-id="6b32f-113">ユーザーは、`--user-data-dir` コマンド ライン フラグを使用して既定値を上書きできます。</span><span class="sxs-lookup"><span data-stu-id="6b32f-113">A user can use the `--user-data-dir` command-line flag to override the default.</span></span>

<span data-ttu-id="6b32f-114">現在の**プロファイル パス**を確認するには、**[バージョン情報]** ページを開きます (「edge://version」と入力します)。</span><span class="sxs-lookup"><span data-stu-id="6b32f-114">To view the current  **Profile path**, open the **About version** page (type "edge://version").</span></span> <span data-ttu-id="6b32f-115">**プロファイル パス**には *C:\Users\\&lt;現在のユーザー&gt;\AppData\Local\Microsoft\Edge\User Data\Default* という形式を使用します。</span><span class="sxs-lookup"><span data-stu-id="6b32f-115">The **Profile path** follows this format: *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\User Data\Default*.</span></span>

### <span data-ttu-id="6b32f-116">パス変数を使用するためのガイダンス</span><span class="sxs-lookup"><span data-stu-id="6b32f-116">Guidance for using path variables</span></span>

<span data-ttu-id="6b32f-117">パスに変数を使用する前に、次のガイダンスをご確認ください。</span><span class="sxs-lookup"><span data-stu-id="6b32f-117">Review the following guidance before using variables for paths.</span></span>

- <span data-ttu-id="6b32f-118">Microsoft Edge によってさまざまなデータが格納されるパスを決定するすべてのポリシーは、プラットフォームによって異なります。</span><span class="sxs-lookup"><span data-stu-id="6b32f-118">All policies that involve paths where Microsoft Edge stores different data are platform dependent.</span></span> <span data-ttu-id="6b32f-119">これらのポリシーには、特定のプラットフォームでのみ使用できるものと、すべてのプラットフォームで使用できるものがあります。</span><span class="sxs-lookup"><span data-stu-id="6b32f-119">Some of these policies are available only on specific platforms, but others can be used on all platforms.</span></span>
- <span data-ttu-id="6b32f-120">さまざまな場所から起動されるアプリケーションによって発生するエラーを回避するには、必ず絶対パスを指定してください。</span><span class="sxs-lookup"><span data-stu-id="6b32f-120">To avoid errors caused by applications starting from different locations on different occasions, make sure that paths are absolute.</span></span>
- <span data-ttu-id="6b32f-121">変数はすべて、1 つのパス内で 1 回のみ指定できます。</span><span class="sxs-lookup"><span data-stu-id="6b32f-121">Every variable can occur only once in a path.</span></span> <span data-ttu-id="6b32f-122">ほとんどの場合、変数は絶対パスに解決されるため、変数を使用するにはこの方法を使用する必要があります。</span><span class="sxs-lookup"><span data-stu-id="6b32f-122">For most of them, this is the only meaningful way to use variables, because they resolve to absolute paths.</span></span>
- <span data-ttu-id="6b32f-123">ほとんどのポリシーでは、パスが存在しなければ作成されます (既存の環境で可能な場合)。</span><span class="sxs-lookup"><span data-stu-id="6b32f-123">Almost all policies will create the path if it doesn't exist (if possible in the existing circumstances).</span></span>
- <span data-ttu-id="6b32f-124">Microsoft Edge のバージョン/チャネルによってフォルダー構造の処理方法が異なるため、ポリシーによっては、ネットワークの場所を使用すると、予期しない結果を招く場合があります。</span><span class="sxs-lookup"><span data-stu-id="6b32f-124">Using network locations for some policies can lead to unexpected results due to differences in how different versions/channels of Microsoft Edge handle the folder structure.</span></span>
